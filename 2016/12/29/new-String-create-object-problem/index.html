new String() create object problem

2016年12月29日

String s=new String(“hello”);创建了几个String对象

1.首先来澄清几个概念:
Stack: 栈,由JVM分配区域,用于保存现成执行的动作(断点)和数据引用.
Heap: 堆,由JVM分配的,用于存储对象等数据的区域.
Constant pool: 常量池,在堆中分配出来的一块存储区域,用于存储显式的String,float,integer.这是一个特殊的共享区域,可以在内存中共享的不经常改变的东西.

  • String a = “abc”; 1
  • String b = “abc”; 2
    使用这种方式,可在在常量池创建对象,可以提高运行速度.1执行后,2再次执行时,检测到常量池存在就不会创建对象了.

上面2句代码:Class被ClassLoader加载,”hello”被作为常量读入到常量池,创建了一个共享的”hello”对象,然后调用
new String(“hello”)的时候,会在heap生成一个对象. 执行2代码时,只用生成一个对象.
综上,String c = new String(“hello”); //创建两个对象,一个引用
String a = “xyz”; //创建一个对象,一个引用
